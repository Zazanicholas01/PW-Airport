UnityWsClientMulti SCRIPT

    - Unity WebSocket Client --> Talk to Server Python --> Serve Position Queries --> Execute Commands
    - Serializable Class --> Define JSON of Messages
        - Query get.position --> ResponseMsg --> {result: Position, t_sim}
        - Command set.route --> EventMsg ack/error
        - Command speed.set --> """
        - Event route.complete --> EventMsg from Unity to Python
    
    - ON START --> Scan the Scene --> Search SimObjects
        - _byId --> Target ID --> GameObject
        - _follow --> Target ID --> Route Follower
        - Checks RouteComplete --> Send event route.complete to Python
    
    - CONNECTION & RECONNECT
        - ConnectLoop() --> Create Client WebSocket --> To Python Ws URL --> Auto Reconnect on errors
        - Run in Background --> Keep networking active
    
    - RECEIVING MESSAGES
        - ReceiveLoop() --> Assemble Message Frames --> Call HandleMessage(json)
        - Type - Query && Query - get.position --> Target by ID --> Reply transform.position --> Or error not found
        - Type - Command speed.set --> Check RouteFollower Exists --> Update Speed --> Optional Acc/Dec --> Send command.ack / command.error
        - Type - Command set.route --> Validate Target & Waypoints List --> Convert JSON to Vector3[] --> Call RouteFollower.StartRoute(wps, speed) --> Send command.ack / command.error
    
    - SENDING MESSAGES
        - Send(json) --> Write UTF-8 Text Frame --> Uses Serialized Classes
    
    - CONFIGURATION
        - PythonWsUrl --> Set ws:// Endpoint for Python Server
        - ReconnectDelaySec --> Reconnect Backoff
    

UNITY & PYTHON COMMUNICATION

    - Python Sends --> Query get.position / Commands set.route && speed.set
    - Unity Responds --> Positions / Command route.complete

RouteFollower SCRIPT

    - PURPOSE --> Move GameObject --> Following a Sequence of Waypoints --> Notifying Route Completion

    - MOTION & SPEED
        - Constants --> speedMetersPerSec / minSpeed / maxSpeed
        - Variables --> _targetSpeed / _currentSpeed
        - Acceleration --> accelUp / accelDown
        - Each Update --> Speed changes by +- accel * deltaTime --> Clamp to Normalize
    
    STEERING & ORIENTATION
        - orientToVelocity --> Rotate Object to Face Movement Direction
        - baseEulerOffset --> Fixed Rotation --> Preserve Model Orientation
    
    WAYPOINTS & PROGRESS
        - Internal State --> _wps (Vector3[]) --> _idx (Current Index) --> _active Flag
        - Vector3.MoveTowards --> Move to Waypoint by speed * dt
    
    COMPLETION
        - Last Waypoint --> _active to False --> Call OnRouteComplete()
    
    PUBLIC API
        - StartRoute --> Start / Restart Following --> Manages Speed
        - SetTargetSpeed --> Change Target Speed
        - StopNow() --> Set all Speed to 0
        - CancelRoute() --> Stop Following --> Clear Route

SensorUIController SCRIPT

    PURPOSE --> HTTP Endpoint Once per second --> Parse JSON Sensor Data --> Update On-Screen UI

    DATA MODEL
        - SensorData --> {light_value, led_state, datetime}
    
    CONFIG & UI
        - serverUrl --> Defines the HTTP Endpoint
        - UI Fields --> 3 Texts for every Sensor Data
        - Optional --> LED Indicator Image --> Light Slider
    
    LIFECYCLE & FLOW
        - Start --> Invoke Repeating --> Schedules Polling Every x seconds
        - FetchData --> Starts GetSensorData()
        - GetSensorData() --> Unity Web Request (GET) --> Send Request
        - ON Success --> Parse JSON to SensorData --> Update UI
        - On Error --> Log "Errore Connessione"
    
    UI UPDATES
        - Set Text --> Light / State / Datetime Texts
        - Update Light Slider.value (0-1023 for Analog)
        - Set LED Indicator.color to Green (ON) or Red (OFF)
        - Logs Summary
